# Site Concept

## Core Idea

This is not another chatbot and not another SaaS.

The core idea:

```text
Chat is for intent.
Apps are for memory, structure, visualization, and action.
AI builds the apps.
```

In Russian:

```text
Чат нужен, чтобы выразить намерение.
Приложения нужны, чтобы хранить, структурировать и показывать данные.
ИИ создает эти приложения под пользователя или компанию.
```

AI needs somewhere to store data, structure it, visualize it, and make it useful over time. A chat thread is not enough for that.

Chats are good for expressing what the user wants:

- "I want to fix my finances."
- "I want to track my health."
- "I want to manage customer requests."
- "I want to digitize my company."

But chat is a weak long-term interface:

- data gets buried in messages
- tables are awkward
- dashboards are limited
- workflows are hard to operate
- state is hard to inspect
- it is difficult to return to the same system weeks later
- there is no dedicated UI for the user's actual process

The answer is not to make chat more complicated.

The answer is for AI to create applications.

## Product Positioning

The product creates software around a person or company.

The user describes what they need in natural language. AI creates apps, databases, interfaces, dashboards, and workflows for that need.

Those apps then become the structured memory and operating surface for the user's life or company.

## Audience Switch

The site should have a clear switch:

```text
For You | For Your Company
```

or in Russian:

```text
Для тебя | Для твоей компании
```

This matters because the same product solves the same structural problem in two contexts.

For an individual, life is fragmented across habits, health, learning, finances, notes, goals, and projects.

For a company, operations are fragmented across CRM, finance, support, inventory, HR, documents, spreadsheets, and internal tools.

In both cases, the user does not need another generic SaaS. They need a system that matches how they actually live or work.

## For You

AI creates personal applications for different parts of life:

- health
- habits
- sleep
- nutrition
- workouts
- psychology and reflection
- learning
- goals
- personal finances
- documents
- projects
- relationships
- personal knowledge base


## Product Mechanics

At a high level:

```text
User describes goal or process
  -> AI designs the data model
  -> AI creates the app
  -> AI deploys it
  -> user starts using it
  -> data accumulates
  -> AI improves and connects apps over time
```

The apps are not static templates. They are living systems:

- generated from user needs
- backed by real databases
- editable by AI
- connected to other apps
- visualized through custom interfaces
- controlled through chat

## Key Differentiation

Most AI products stop at conversation.

This product turns conversation into software.

Most SaaS products force users into predefined workflows.

This product creates workflows around the user.

Most no-code tools require users to build the system manually.

This product lets the user describe the system, while AI builds and evolves it.
